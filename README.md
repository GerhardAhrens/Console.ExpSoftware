# Gedanken zur heutigen Softwareentwicklung

Softwareentwicklung ist heute teuer und aufwendig. Das fängt mit der Betrachtung von Sicherheitsaspekten an, und geht weiter in die Auswahl von passenden NuGet-Paketen bis hin zur Erhaltung und Funktionfähigkeit über den Zeitraum der Verwendung.

Ein Gedanke dazu ist eine **Hybride Softwareentwicklung**, wobei es nicht um die Methodik einer Entwicklung geht, sondern um die konkrete Programmierung.

Hinter dem einer Einsatz **Hybride Softwareentwicklung** geht es in erster Linien darum, mit den zur Verfügung stehenden Mittel Software zu erstellen, die einer gewünschten Anforderung entspricht. Es sollen so wenig wie möglich Einschrägungen in der Entwicklung von Software vorhanden sein.

- Native Programmierung
- Templates
- Source Generatoren
- Open Source Pakete
- Kauf-Bibliotheken 

sind hierbei geeignete Mittel.
Nun kommt das **ABER**. Wie Eingangs erwähnt gibt es natürlich auch Anforderungen, die in Betracht gezogen werden müssen.
- Wartbarkeit
- Sicherheit
- Nutzungsdauer
- Nachvollziebarkeit

Hier sieht man schnell, dass die schöne Welt *"alles ist möglich"* schnell wieder zusammenschrumpft.
